Producer–Consumer Problem Using Semaphores

Intoduction

  • The Producer–Consumer Problem is a classic synchronization problem in Operating Systems.
  • It deals with multiple processes sharing a common buffer.
  • The producer creates data and puts it into the buffer.
  • The consumer takes data from the buffer and processes it.
  • Proper synchronization is required to avoid race conditions.

Why Synchronization is Needed

  • If the producer adds data when the buffer is full, data may be lost.
  • If the consumer removes data when the buffer is empty, it causes errors.
  • Semaphores are used to control access and maintain synchronization.
  • wait(S): Decreases the semaphore value by 1. If the value is ≤0, the process waits.
  • signal(S): Increases the semaphore value by 1, potentially unblocking waiting processes.

while(S <= 0); // busy waiting
S--;
}
signal(S){
S++;

Semaphores (Brief Overview) & their Variables

A semaphore is a synchronization tool used to manage access to shared resources.
Types of Semaphores Used:

  • mutex(S) → Ensures mutual exclusion
  • empty(E) → Counts empty slots in buffer
  • full(F) → Counts filled slots in buffer

Assume: Buffer size = N

Semaphore Initial Value Purpose
mutex 1 Protects critical section empty
empty N Number of empty slots
full 0 Number of filled slots

Producer Process (Logic)

  • Wait if buffer is full
  • Enter critical section
  • Produce item
  • Place item in buffer
  • Exit critical section
  • Signal consumer

void producer()
{
while (true)
{
produce(); // produce an item

wait(empty); // wait if buffer is full
wait(mutex); // enter critical section

append(); // add item to buffer

signal(mutex); // exit critical section
signal(full); // increment full slots
{
}

Consumer Process (Logic)

  • Wait if buffer is empty
  • Enter critical section
  • Remove item from buffer
  • Consume item
  • Exit critical section
  • Signal produce

void consumer()
{
while (true)
{
wait(full); // wait if buffer is empty
wait(mutex); // enter critical section

remove(); // remove item from buffer

signal(mutex); // exit critical section
signal(empty); // increment empty slots

consume(); // consume item
{
}

Real-Life Examples

  • Print queue (printer as consumer)
  • Producer apps generating logs, consumer storing them
  • Online streaming (buffering)
  • Operating System I/O management

Advantages

    ✔ Prevents race conditions
    ✔ Ensures data consistency
    ✔ Efficient CPU utilization
    ✔ Widely used in real systems

Conclusion

The Producer–Consumer Problem shows how processes are synchronized while sharing a common buffer. Semaphores provide a safe and efficient way to control access and avoid race conditions. It is a fundamental concept in Operating Systems.
